# Guia de Desenvolvimento e Gerência de Configuração

Bem-vindo ao repositório de **documentação de Gerência de Configuração de Software** do projeto.  
Este repositório reúne os documentos que definem os processos utilizados para controle de mudanças, organização do código e colaboração entre os membros da equipe.

Os documentos aqui presentes descrevem as práticas adotadas para gerenciamento de branches, padronização de commits e classificação de issues, garantindo um processo de desenvolvimento organizado e rastreável.

---

# Modelo de Branches — GitHub Flow

O projeto adota o **GitHub Flow** como modelo de gerenciamento de branches.

Esse modelo define um fluxo de trabalho simples e eficiente, no qual a branch principal (`main`) mantém sempre uma versão estável do projeto, enquanto novas funcionalidades, correções de bugs ou melhorias são desenvolvidas em branches separadas.

Após o desenvolvimento, as alterações são revisadas por meio de **Pull Requests** antes de serem integradas à branch principal. Esse processo garante maior controle sobre as mudanças e melhora a colaboração entre os membros da equipe.

Mais detalhes sobre o modelo adotado podem ser encontrados no documento:

[Branch Model — GitHub Flow](https://github.com/Artrite-Fit/docs/blob/docs/config-management/Ger%C3%AAncia%20de%20Configura%C3%A7%C3%A3o%20de%20Software/branch_model.md)

# Padrão de Commits — Guia de Estilo

Para garantir clareza e rastreabilidade no histórico do projeto, utilizamos um **padrão de mensagens de commit baseado em commits semânticos**.

Esse padrão define uma estrutura para as mensagens de commit, permitindo identificar facilmente:

- o tipo de alteração realizada
- a parte do sistema afetada
- a finalidade da modificação

O uso desse padrão facilita a compreensão do histórico do projeto, melhora a organização do repositório e auxilia no gerenciamento de mudanças.

O guia completo pode ser consultado no documento:

[Guia de Estilo para Mensagens de Commit](https://github.com/Artrite-Fit/docs/blob/docs/config-management/Ger%C3%AAncia%20de%20Configura%C3%A7%C3%A3o%20de%20Software/commit-style-guide.md)

# Modelo de Severidade e Prioridade

Para apoiar o processo de **gerenciamento de mudanças e resolução de problemas**, o projeto utiliza um modelo de classificação de **severidade e prioridade de issues**.

Esse modelo permite avaliar:

- o impacto de um problema no funcionamento do sistema (**severidade**)
- a urgência e importância da resolução (**prioridade**)

A classificação ajuda a equipe a priorizar as tarefas de forma adequada e direcionar os esforços para as correções mais críticas ou melhorias necessárias.

O documento completo com os critérios de classificação pode ser consultado em:

[Modelo de Severidade e Prioridade](https://github.com/Artrite-Fit/docs/blob/docs/config-management/Ger%C3%AAncia%20de%20Configura%C3%A7%C3%A3o%20de%20Software/severity-priority-model.md)

# Objetivo da Documentação

A documentação de Gerência de Configuração tem como objetivo estabelecer padrões claros para:

- organização do repositório
- controle de mudanças
- padronização de commits
- gerenciamento de issues
- colaboração entre os membros da equipe

A adoção dessas práticas contribui para um processo de desenvolvimento mais organizado, transparente e eficiente.
